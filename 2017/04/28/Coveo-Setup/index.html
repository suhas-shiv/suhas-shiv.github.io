Coveo Setup

Posted on April 28, 2017

CES Installation Notes

Version to be installed for sitecore 8.1

Configuration of Accounts while CES Installation:

  • You can select your Domain account as user for indexes (A user on CES server that manages all indexes, in this case CES server is your local server)
  • For Rabbit MQ, you can use default guest user if you don’t have any other account.
  • For local purpose, admin service security is left disabled and made it available on port 80, left all the ports same since there are no conflicts locally on this port.

Click Next and Installation will start; it should take like about 5 -10 mins.

Verify all Coveo Windows Services running

Coveo Search API install

Download version of Coveo Search API corresponding to your CES version.
https://developers.coveo.com/display/public/SitecoreV4/Installing+Coveo+Enterprise+Search+%28CES%29+On-Premises
Note - Select “This installation is part of a Sitecore Integration” option.

Generate a random security token – Take a note of this secrete key
On local, let Coveo install with its default location. Once, installation is complete you will see below, you can choose to click finish

All configurations and files related to Coveo Search API will be installed below –

New service is added for search api, it will not be running because we did not set up Coveo for Sitecore yet and also config.yml should be updated in later step before this gets started.

Notes for Coveo for Sitecore Installation –

  • Get the corresponding zip of package file from Coveo site for Sitecore 8.1
  • Install the package using Installation wizard

  • Follow instructions here, paying attention to below bullet points– (Ensure you select free so that you don’t have to enter license or key)
    https://developers.coveo.com/display/public/SitecoreV4/Installing+Coveo+for+Sitecore

  • Use default values while installation for local

  • Select -> Index only sitecore items, we don’t need to index the whole rendered html as it will be expensive and indexes will be huge, for our business requirements we would only need to index sitecore items.
  • Check box auto rename the config files to enable Coveo.
  • Open config.yml in C:\Program Files\Coveo Search API 8 and make sure URL and other things listed are right
  • Restart Coveo Search API service from windows services
  • Rebuild your indexes
    That’s it, you should be done. Open Coveo Diagnostics page from control panel to ensure all is well on this installation.
    Final Diagnostics – (Control Panel -> Coveo Section -> Diagnostics)

All should be green.
Build Coveo indexes in index manager. You are all set.
